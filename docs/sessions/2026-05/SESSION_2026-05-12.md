### 5. Seed Prisma Sécurisé
**Fichier modifié** : `prisma/seed.ts`

**Problème identifié** :
- Ancien seed utilisait `deleteMany()` → supprimait TOUT les services
- Services ajoutés via le dashboard après mars risquaient d'être perdus

**Solution** :
- `deleteMany()` remplacé par `upsert()`
- Si service existe (même ID) → mis à jour
- Si service n'existe pas → créé
- Services dashboard (IDs différents) → jamais touchés

**Résultat** :
- 20 services synchronisés en production
- Aucune donnée perdue
